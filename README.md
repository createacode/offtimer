# offtimer
一款色泽艳丽，简洁好用的定时关机程序，可以实现win10/win11的定时关机，清理进程等功能，支持开机自启，记忆定时时间，采用C++开发，使用MSYS2 UCRT64编译。


# 定时关机程序（Windows版）使用说明

## 一、程序简介

本程序是一个定时关机辅助工具，允许您设定一个时间，到点后自动清理非系统进程并强制关机。同时提供立即关机、一键清理内存、开机自启等功能。程序运行时以托盘图标常驻后台，不占用任务栏。

## 二、安装与运行

### 方法一：直接运行

- 双击 `APP121411.exe` 即可打开主窗口。
- 若需要开机自启，请勾选“开机自启”复选框（需要管理员权限写入注册表，首次运行可能弹出 UAC 提示）。

### 方法二：静默启动（开机自启）

- 程序以 `--silent` 参数启动时，不会显示主窗口，仅显示托盘图标。
- 可通过双击托盘图标打开主界面。

## 三、界面说明

### 主窗口布局

| 区域 | 内容 |
|------|------|
| 日志显示区 | 显示所有操作记录，带时间戳，自动滚动到底部 |
| 定时关机时间 | 小时/分钟输入框（支持循环滚动），以及“开机自启”复选框 |
| 按钮区 | 定时关机、立即关机、一键清理 |

### 托盘图标

- 右键单击托盘图标：弹出菜单（打开主界面 / 退出程序）。
- 左键双击托盘图标：显示/隐藏主窗口。

## 四、功能操作指南

### 1. 设定定时关机

1. 在“时”和“分”输入框中设定目标时间（支持键盘上下键或鼠标滚轮，边界自动循环）。
2. 点击“定时关机”按钮。
3. 程序会弹出提示框确认设定，并在日志中记录。
4. 到点后，会弹出确认对话框：“定时关机时间已到，是否立即关机？若不操作，2分钟后将自动关机。”
   - 点击“是”：立即执行关机。
   - 点击“否”：取消本次定时关机。
   - 不操作：30秒后对话框自动关闭，2分钟后自动关机。

### 2. 立即关机

1. 点击“立即关机”按钮。
2. 弹出确认对话框：“系统将清理进程后强制关机，是否继续？”
   - 点击“是”：立即清理进程并关机。
   - 点击“否”：取消操作。
   - 不操作：2分钟后自动确认并关机。

### 3. 一键清理

1. 点击“一键清理”按钮。
2. 程序将终止所有非系统关键进程（白名单内的进程不会被终止）。
3. 清理完成后，日志中显示释放的内存大小和当前空闲内存占比。
4. 弹出消息框提示清理结果。

### 4. 开机自启设置

- 勾选“开机自启”复选框：程序将在下次开机时自动静默启动（无主窗口，仅托盘）。
- 取消勾选：删除开机自启注册表项。

## 五、日志查看

- **窗口日志**：主窗口上半部分显示所有操作记录，可滚动查看。
- **文件日志**：程序所在目录下的 `日志/` 文件夹中，每天（每次启动）生成一个独立的文本文件，命名格式如 `20260315_143022_A3B9_日志.txt`。

## 六、配置文件

- 程序会在自身所在目录生成 `config.ini` 文件，用于保存：
  - 上次设定的定时关机时间
  - 开机自启复选框状态
  - 当前激活的定时任务
- 删除该文件后，程序将恢复默认设置（定时时间 22:00，不开机自启）。

## 七、常见问题

### 1. 程序无法启动，提示缺少 DLL

- 请确保所有依赖 DLL 与主程序在同一目录（参考开发文档中的依赖收集方法）。

### 2. 关机后电脑未断电（指示灯亮、风扇转）

- 此问题多见于某些品牌电脑（如戴尔、联想），可能与快速启动或电源管理驱动有关。本程序已采用强制断电关机（`EWX_POWEROFF | EWX_FORCE`），若仍出现，请尝试：
  - 关闭 Windows 快速启动（控制面板 → 电源选项 → 选择电源按钮的功能 → 更改当前不可用的设置 → 取消“启用快速启动”）。
  - 更新主板和电源管理驱动。

### 3. 定时关机没有执行

- 检查程序是否正在运行（托盘图标是否存在）。
- 查看日志窗口或日志文件，确认是否有“定时关机条件触发”的记录。
- 确保系统时间准确。

### 4. 开机自启没有生效

- 检查是否被安全软件拦截（如 Defender、360）。
- 以管理员身份运行一次程序，重新勾选“开机自启”。
- 手动检查注册表：`HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run`，确认存在 `APP121411` 键值，指向正确的 exe 路径并带有 `--silent` 参数。

### 5. 程序被 Windows Defender 报告为病毒

- 本程序为开源项目，不含任何恶意代码。由于涉及进程终止和关机操作，部分杀毒软件可能误报。请将程序目录添加到 Defender 排除列表，或暂时关闭实时保护。

## 八、卸载程序

由于本程序为绿色软件，无需安装。卸载方法：
1. 通过托盘右键菜单退出程序。
2. 删除程序所在文件夹及 `config.ini`、`日志/` 目录。
3. 手动删除注册表项：`HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run` 下的 `APP121411`。

## 九、技术支持

- 本程序为个人开发，如有问题请在项目发布渠道反馈。
- 源代码已提供，允许二次开发，但请保留版权信息。

# 定时关机程序（Windows版）开发文档

## 一、项目概述

本项目实现了一个基于 Qt6 的 Windows 桌面应用程序，提供定时关机、立即关机、进程清理等功能，支持系统托盘常驻运行。程序使用 MSYS2 UCRT64 环境编译，最终生成 `APP121411.exe`。

## 二、项目结构

```
D:\Demo\demo37/
├── demo37.pro          # Qt 项目文件
├── main.h              # 头文件（类声明、全局函数）
├── main.cpp            # 实现文件
├── resources.qrc       # 资源文件（嵌入 app.ico）
├── version.rc          # 版本资源文件（包含图标和版本信息）
├── app.ico             # 程序图标（需用户提供）
└── 日志/               # 运行时生成的日志文件夹（自动创建）
```

## 三、编译环境

- **操作系统**：Windows 10/11
- **编译环境**：MSYS2 UCRT64
- **Qt 版本**：Qt 6.x（建议 6.5+）
- **编译器**：MinGW-w64 GCC（UCRT64 版）
- **构建工具**：qmake, make

## 四、环境搭建与编译步骤

### 1. 安装 MSYS2 及 Qt6

```bash
# 安装 MSYS2（从官网下载安装包）
# 打开 MSYS2 UCRT64 终端，执行以下命令更新并安装必要组件
pacman -Syu
pacman -Su
pacman -S mingw-w64-ucrt-x86_64-qt6-base \
          mingw-w64-ucrt-x86_64-qt6-tools \
          mingw-w64-ucrt-x86_64-gcc \
          mingw-w64-ucrt-x86_64-make
```

### 2. 获取源码

将提供的所有源文件放入 `D:\Demo\demo37` 目录，并确保 `app.ico` 存在。

### 3. 编译

```bash
cd /d/Demo/demo37
qmake demo37.pro
make
```

编译成功后，可执行文件位于 `release/APP121411.exe`。

### 4. 收集依赖（分发用）

```bash
mkdir -p deploy
cp release/APP121411.exe deploy/
cd deploy
windeployqt6 APP121411.exe
for f in *.exe *.dll; do
    ldd "$f" 2>/dev/null | grep '/ucrt64/bin/' | awk '{print $3}' | sort -u | while read -r dep; do
        [ -f "$dep" ] && cp -n "$dep" .
    done
done
```

打包整个 `deploy` 文件夹即可在其他 Windows 电脑运行。

## 五、核心功能模块

### 1. 定时关机

- 用户设定时、分，点击“定时关机”后，程序每分钟检查一次当前时间。
- 若当前时间在设定时间前后一分钟内，弹出非模态确认对话框。
- 若用户 30 秒内无操作，对话框自动关闭，2 分钟后自动执行关机（清理进程后强制断电）。
- 若用户点击“是”，立即执行关机；点击“否”，取消本次定时任务。
- 定时任务状态保存在 `config.ini` 中，程序重启后自动恢复。

### 2. 立即关机

- 点击后弹出确认对话框，2 分钟内未点击“是”则自动确认。
- 确认后先清理非系统进程，然后执行强制断电关机。

### 3. 一键清理

- 终止所有非系统关键进程（排除白名单）。
- 输出释放的内存大小和空闲内存占比。
- 详细记录每个被终止的进程名称和 PID。

### 4. 系统托盘

- 程序启动后常驻系统托盘，支持双击显示/隐藏主窗口。
- 右键菜单提供“打开主界面”和“退出程序”。
- 唯一退出方式为托盘右键菜单中的“退出”。

### 5. 开机自启

- 勾选“开机自启”后，程序在当前用户注册表 `Run` 键下添加启动项。
- 启动参数为 `--silent`，实现静默启动（无窗口，仅托盘）。
- 自启状态下，程序会等待 5 秒再初始化，避免系统资源竞争。

### 6. 日志系统

- **窗口日志**：主窗口多行文本框，显示带时间戳的操作记录，自动滚动。
- **文件日志**：保存在程序所在目录的 `日志/` 文件夹下，文件命名格式 `日期_时分秒_随机串_日志.txt`。
- 日志内容包括：程序启动时间、电源离开模式状态、定时设定、进程清理详情、关机执行情况等。

## 六、关键技术实现

### 1. 自定义循环滚动 SpinBox

`DoubleSpinBox` 继承自 `QSpinBox`，重写 `textFromValue` 和 `valueFromText` 实现两位数显示，并通过 `setWrapping(true)` 实现数值循环（23→0, 0→23；59→0, 0→59）。

### 2. 非模态确认对话框 + 延迟关机

- 使用 `QMessageBox` 并设置 `setModal(false)`，避免阻塞主事件循环。
- 同时启动一个 2 分钟的单次定时器，到期后自动执行关机。
- 对话框 30 秒后自动关闭，不影响延迟定时器。

### 3. 强制断电关机

采用三层后备机制：
1. 优先使用 `ExitWindowsEx(EWX_POWEROFF | EWX_FORCE, ...)`。
2. 失败则调用 `InitiateShutdown`。
3. 最后降级使用 `system("shutdown /s /f /t 0")`。

在关机前提升本进程的关机优先级（`SetProcessShutdownParameters`），确保清理代码先于其他进程执行。

### 4. 配置持久化

使用 `QSettings` 以 Ini 格式读写 `exe目录/config.ini`，存储以下字段：
- `ShutdownHour`, `ShutdownMinute`：上次设定的定时时间。
- `AutoStart`：开机自启复选框状态。
- `ScheduledActive`, `ScheduledHour`, `ScheduledMinute`：当前激活的定时任务。

### 5. 单实例控制

通过 `QSharedMemory` 创建共享内存段 `"APP121411_SINGLE_INSTANCE_KEY"`，若已存在则提示并退出新实例。

### 6. 进程清理白名单

排除以下系统关键进程，避免系统崩溃：
```
explorer.exe, svchost.exe, csrss.exe, winlogon.exe, services.exe,
lsass.exe, wininit.exe, system, smss.exe, dwm.exe, fontdrvhost.exe,
taskhostw.exe, sihost.exe, ctfmon.exe, audiodg.exe, spoolsv.exe,
msmpeng.exe, SearchIndexer.exe, WUDFHost.exe, WmiPrvSE.exe,
dllhost.exe, RuntimeBroker.exe, SettingSyncHost.exe, LogonUI.exe
```

## 七、自定义控件说明

### `DoubleSpinBox`

- 继承自 `QSpinBox`，重写 `textFromValue` 返回两位数字符串（如 `"08"`）。
- 重写 `valueFromText` 将字符串转回整数。
- 在 `setupUI` 中调用 `setWrapping(true)` 实现循环滚动。

## 八、常见问题及解决

### 1. 编译错误：缺少头文件

- 确保 `.pro` 文件中 `QT += core widgets gui` 已添加。
- 在 MSYS2 UCRT64 中运行 `pacman -S mingw-w64-ucrt-x86_64-qt6-base` 重新安装。

### 2. 运行时缺少 DLL

- 使用 `windeployqt6` 和上述依赖收集脚本，将所有依赖放入同目录。

### 3. Windows Defender 误报

- 添加版本资源文件（`version.rc`），包含公司名、文件描述等信息，降低误报概率。
- 若仍被拦截，请将程序目录添加到 Defender 排除列表。

### 4. Win11 下 SpinBox 按钮变为水平排列且不灵敏

- 在 `main` 函数中设置全局样式：`a.setStyle(QStyleFactory::create("windowsvista"));`
- 同时为 `hourSpin` 和 `minuteSpin` 添加详细样式表，强制上下按钮独立绘制（已在代码中实现）。

### 5. 关机后电脑未断电

- 已使用 `EWX_POWEROFF | EWX_FORCE` 强制断电，若仍存在，请更新主板驱动或关闭 Windows 快速启动。

## 九、API 参考

### 全局函数

- `bool PerformForceShutdown()`：执行强制断电关机，返回成功与否。
- `MemInfo killNonSystemProcesses()`：清理非系统进程，返回内存信息。
- `void logMessage(const QString &msg, QTextEdit *textEdit)`：写入日志（窗口和文件）。

### MainWindow 类

- `void onTimedShutdown()`：设定定时关机。
- `void onImmediateShutdown()`：立即关机。
- `void onCleanup()`：一键清理。
- `void onTimerCheck()`：每分钟检查定时任务。
- `void loadSettings()` / `saveSettings()`：配置读写。
